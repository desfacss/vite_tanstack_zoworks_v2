## 1. ONDC (Open Network for Digital Commerce) – India's National Standard

**ONDC is the primary standard for the Indian market** and represents a fundamental shift from platform-centric to network-centric commerce .

### What It Is:
ONDC is a government-backed initiative to democratize digital commerce by creating an open, interoperable network where buyers and sellers can transact regardless of which platform they use .

### Technical Foundation:
- **Built on Beckn Protocol** – an open protocol that standardizes communication between different entities in the digital commerce ecosystem 
- **Defines complete transaction lifecycle**: Discovery → Order → Fulfillment → Post-Fulfillment 
- **Supports both B2B and B2C retail** with separate specification tracks 

### API Specifications Cover:
- **Buyer App APIs**: Product discovery, ordering, post-purchase support 
- **Seller App APIs**: Inventory management, order processing, fulfillment 
- **Key transaction flows**: RFQ (Request for Quote), Non-RFQ, prepaid payments, cancellation features 

### For Your Implementation:
- Your catalog schema already supports many ONDC requirements (multi-tenant, location-based, variants)
- You'll need to implement the Beckn protocol APIs for discovery, order, and fulfillment phases
- ONDC provides white-label buyer and seller app SDKs on GitHub 


## 2. Google's Universal Commerce Protocol (UCP) – Global Discovery & Orchestration

**UCP is Google's open-source specification for agent-driven commerce experiences** across Google surfaces (Search, AI Mode, Gemini) .

### What It Does:
UCP defines a common language for AI systems to interact with merchants across the entire shopping journey – from discovery through checkout and order management .

### Core Capabilities:

| Capability | Description |
|------------|-------------|
| **Checkout** | Native or embedded checkout experiences with dynamic pricing and tax calculation; merchants remain Merchant of Record  |
| **Identity Linking** | OAuth 2.0 standards for secure authorization without sharing credentials  |
| **Order Management** | Webhooks for order confirmations, status updates, shipment tracking, and returns  |

### Schema Requirements (Critical for Your Catalog):
According to UCP specifications, your product data must include :

| Field | Priority | Purpose |
|-------|----------|---------|
| **GTIN** (Global Trade Item Number) | **Critical** | The only global "primary key" for physical goods; required for agent interoperability  |
| **shipping_weight** | Critical | Accurate landed cost and shipping promises  |
| **certification** | Compliance | Registry-backed compliance claims (EPREL, etc.)  |
| **product_id_web / product_id_store** | Critical | Split identifiers when price/availability differs by channel  |
| **return_rate** | Critical | Merchant quality and risk signal for AI agents  |
| **auto_pricing_min_price** | Critical | Margin protection floor for automated negotiations  |

### For Your Implementation:
- Your schema has `short_code` and `external_id` fields – you must populate these with **GTIN/EAN/UPC** to be UCP-compliant
- Add `shipping_weight`, `return_rate`, and `certification` fields to your offerings table
- Implement OAuth 2.0 for identity linking
- Set up webhook endpoints for order status updates


## 3. OpenAI's Agentic Commerce Protocol (ACP) – Conversational Checkout

**ACP is an open standard for AI-led commerce**, co-developed by OpenAI and Stripe, enabling purchases directly within AI assistants like ChatGPT .

### What It Does:
ACP enables programmatic exchanges between buyers, AI agents, merchants, and payment providers to complete purchases without traditional website visits .

### Three Core Flows :

| Flow | Description |
|------|-------------|
| **Product Feed** | Live, structured product data including descriptions, media, pricing, availability, fulfillment options  |
| **Agentic Checkout** | Standardized specification allowing agents to render checkout within their own interfaces  |
| **Delegated Payment** | Secure mechanism for agents to share payment details via single-use, time-bound tokens  |

### ACP-Specific Schema Requirements :

| Field | Priority | Purpose |
|-------|----------|---------|
| **enable_checkout** | Critical | Hard gate to block purchases for B2B, regulated products  |
| **return_rate** | Critical | Decimal fraction (e.g., 0.05 = 5%) – risk signal for AI agents  |
| **popularity_score** | New | Integer (1-5) derived from sales velocity  |
| **q_and_a** | Growth | Preemptive answers to product questions  |
| **relationship_type** | Growth | Semantic linking for substitution/cross-sell  |

### For Your Implementation:
- Add `enable_checkout` boolean to offerings table
- Implement ACP's delegated payment token flow (Razorpay supports this)
- Create structured product feeds (JSON/XML) for agent consumption
- Set up Q&A fields for flagship products


## Key Differences Between ACP and UCP

| Dimension | OpenAI ACP | Google UCP |
|-----------|------------|------------|
| **Philosophy** | Agent-first – designed for AI-initiated transactions  | Merchant-first – makes commerce systems legible across the web  |
| **Discovery** | Feed-first: merchants push product feeds to agents  | Schema + Feed: web pages expose agent-readable structured data  |
| **Authorization** | Delegated Payment – single-use tokens  | Identity Linking – OAuth 2.0  |
| **Checkout** | Agent-driven inside conversation UI  | Platform-mediated with native or embedded flows  |
| **Order Management** | Agent reflects states; merchant owns post-purchase  | Full access via account-linked APIs and webhooks  |
| **Target Surfaces** | ChatGPT, conversational assistants  | Google Search, AI Mode, Gemini, Shopping  |


### Implementation Roadmap:

1. **Phase 1 – Foundation (ONDC)**
   - Implement Beckn protocol APIs
   - Set up catalog caching per ONDC specifications 
   - Support RFQ and cancellation flows 

2. **Phase 2 – Discovery (UCP)**
   - Add GTIN to all products (non-negotiable for visibility) 
   - Implement structured data (Schema.org) on product pages
   - Set up Google Merchant Center feed with UCP attributes
   - Implement OAuth 2.0 for identity linking

3. **Phase 3 – Transactional (ACP)**
   - Create structured product feeds for AI agents
   - Implement delegated payment tokens
   - Set up Q&A knowledge base for products
   - Enable checkout gate control per product

### Critical Success Factor: GS1

**GS1 is the foundational layer** that makes both UCP and ACP possible . Without proper GTINs (EAN/UPC), your products are invisible to the global graph and agents cannot reliably identify them .
